Practical 1: Creating a Forensic Image

Aim: Creating a Forensic Image using FTK Imager

  • Creating a Forensic Image
  • Checking the Integrity of Data
  • Analyzing a Forensic Image

Part A: Creating a Forensic Image

In this practical, we will use FTK Imager to create images of the evidence.

  1. Click File, and then Create Disk Image, or click the corresponding button on the toolbar.
  2. Select the source evidence type you want to image and click Next.
  3. Select the source evidence file with its path.
  4. Click on Add to set the image destination.
  5. In the Image Destination Folder field, type the path where you want to save the image file, or click Browse to find the location.
  6. In the Image Filename field, specify a name for the image file (without a file extension). After setting the path, click Finish.
  7. Click Start to begin the image creation process.
  8. After the image is created, click Image Summary to view detailed information, including MD5 and SHA1 checksums.

Part B: Analyzing a Forensic Image

  1. Click on Add Evidence Item to add evidence from a disk, image file, or folder.
  2. Select the source evidence type as Image File.
  3. Browse to and open the evidence image file you created.
  4. Select Evidence Tree from the navigation pane to browse and analyze the contents of the image file.

Part C: Imaging a Physical Drive (e.g., Pen Drive)

  1. Select the source evidence type. Choose Physical Drive and proceed.
  2. Select the correct physical drive from the list (e.g., your pen drive) and click Finish.
  3. Add an image destination. For the format, select raw (dd).
  4. Fill in the evidence item information (case number, evidence number, etc.).
  5. Create a destination folder on a different drive to save the image. Paste this location into the destination path field and click Finish.
  6. Click Start and wait for the imaging process to complete.
  7. Upon completion, FTK Imager will display the computed MD5 and SHA1 hash values. You can verify these against the source to ensure the integrity of the image.
  8. You can view and save the Image Summary for your records.

Practical 2: Data Acquisition

Aim: Perform data acquisition using a USB Write Blocker and Encase Imager.

This practical demonstrates how to enable write protection for USB drives via the Windows Registry, effectively turning the system into a software-based write blocker.

Enabling USB Write Protection (Registry Method)

  1. Open the Run dialog box (Windows Key + R), type regedit, and press Enter to open the Registry Editor.
  2. In the Registry Editor, navigate to the following key in the left panel:
    HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control
  3. Right-click on the Control key, select New, and then click on Key.
  4. Name the new key StorageDevicePolicies and press Enter.
  5. Select the newly created StorageDevicePolicies key in the left panel.
  6. Right-click on an empty space in the right panel, select New, and then choose DWORD (32-bit) Value.
  7. Name this new DWORD value WriteProtect.
  8. Double-click on the new WriteProtect value. Change its 'Value data' from 0 to 1 and click OK.
  9. Restart your computer for the changes to take effect.
  10. After restarting, any USB drive connected to the computer will be write-protected. You can read files from it, but you cannot modify, delete, or add new files to the drive.

Acquisition using Encase Imager (General Steps)

After ensuring the source media is write-protected, you can proceed with imaging.

  1. Launch your forensic imaging tool (e.g., Encase Imager).
  2. From the main interface, choose to create a new evidence file or start a new case.
  3. Select the source device you wish to acquire. This will be the write-protected USB drive.
  4. In the 'Create Image' or similar dialog box, click Add to specify the evidence details.
  5. Fill in the case information, such as Case Number, Evidence Number, Examiner Name, and a unique description of the evidence.
  6. Specify the destination path where the forensic image will be saved. Ensure the destination has enough free space.
  7. Select the image format (e.g., E01, Ex01).
  8. Begin the acquisition process. The tool will create a bit-for-bit copy of the source drive and compute hash values (MD5, SHA-1) to verify the integrity of the created image.

Practical 4: Capturing and Analyzing Network Packets

Aim: Capturing and analyzing network packets using Wireshark (Fundamentals)

  • Identify the live network
  • Capture packets
  • Analyze the captured packets

Capturing Packets

  1. To start capturing traffic, click on your network interface (e.g., Wi-Fi, Ethernet).
  2. As soon as you click the interface, Wireshark will begin capturing packets in real time.
  3. Promiscuous Mode: This mode allows your network card to capture all packets on the network segment, not just those addressed to your machine. It is enabled by default in Wireshark.
  4. To verify, go to Capture > Options. Ensure the "Enable promiscuous mode on all interfaces" checkbox is checked at the bottom of the window.

Analyze the Captured Packets

Color Coding: Wireshark uses colors to help you identify different types of traffic at a glance.

  • Light Purple: TCP traffic
  • Light Blue: UDP traffic
  • Black: Packets with errors

You can create custom coloring rules by going to View > Coloring Rules.

Using Display Filters

Apply filters in the bar at the top of the window to narrow down the captured data.

  1. Display packets for a specific IP address (source or destination):
    ip.addr == 192.0.2.1
  2. Display packets from a specific source IP address:
    ip.src == 192.168.10.28
  3. Display packets going to a specific destination IP address:
    ip.dst == 192.168.10.28
  4. Display packets using the HTTP protocol:
    http
  5. Display only HTTP request packets:
    http.request
  6. Display packets using the TCP protocol:
    tcp
  7. Display packets with an HTTP success code (OK):
    http.response.code == 200
  8. Display packets on either TCP port 80 or UDP port 443:
    tcp.port == 80 || udp.port == 443
  9. Display TCP packets containing a specific keyword (e.g., "facebook"):
    tcp contains facebook

Further Analysis

  1. Find Web Server Name: The web server name is often in the HTTP response header. Apply the filter http.response to see all HTTP response packets.
  2. Apply as Column: To easily see all server names, find a packet, expand the HTTP protocol details in the packet details pane, right-click on the "Server" field, and select "Apply as Column".
  3. View Endpoints: To see a summary of all connections, go to Statistics > Endpoints. This will show all IP addresses that communicated during the capture.

Conclusion

We have successfully analyzed the packets provided and solved the questions using Wireshark.

Practical 5: Using Sysinternals Tools for Network Tracking and Process Monitoring

Aim:

  • Check Sysinternals tools
  • Monitor Live Processes
  • Capture RAM
  • Capture TCP/UDP packets
  • Monitor Hard Disk
  • Monitor Virtual Memory
  • Monitor Cache Memory

Check Sysinternals Tools

Windows Sysinternals tools are utilities to manage, diagnose, troubleshoot, and monitor a Microsoft Windows environment. First, download the suite from the official Microsoft website if not already installed.

The tools fall into several categories:

  1. File and Disk Utilities
  2. Networking Utilities
  3. Process Utilities
  4. Security Utilities
  5. System Information Utilities
  6. Miscellaneous Utilities

1. Monitor Live Processes (Tool: Process Monitor - ProcMon)

ProcMon is an advanced monitoring tool for Windows that shows real-time file system, Registry and process/thread activity.

Features to use: Filter, Process Tree, Process Activity Summary, Count Occurrences.

Steps:

  1. Run procmon.exe.
  2. Grant administrative permissions if prompted.
  3. The capture starts immediately. You can use the Filter menu (Ctrl+L) to narrow down the results to a specific process name, PID, etc.
  4. Click Apply and OK to see the filtered processes.

2. Capture RAM (Tool: RAMMap)

RAMMap is an advanced physical memory usage analysis utility.

Steps:

  1. Run RAMMap.exe.
  2. Allow the permissions to view a detailed, color-coded map of how Windows is assigning physical memory.

3. Capture TCP/UDP packets (Tool: TCPView)

TCPView shows detailed listings of all TCP and UDP endpoints on your system, including local/remote addresses and connection states.

Steps:

  1. Run TCPView.exe.
  2. The program will immediately enumerate and display all active TCP and UDP connections.

4. Monitor Hard Disk (Tool: DiskMon)

DiskMon logs and displays all hard disk activity on a Windows system.

Steps:

  1. Run Diskmon.exe as an Administrator.
  2. It will begin logging all disk I/O activity.

5. Monitor Virtual Memory (Tool: VMMap)

VMMap is a process virtual and physical memory analysis utility.

Steps:

  1. Run VMMap.exe.
  2. Select a running process to see a graphical breakdown of its committed virtual memory types and the physical memory assigned to them.

6. Monitor Cache Memory (Tool: CacheSet)

CacheSet is an applet that allows you to manipulate the working-set parameters of the system file cache.

Steps:

  1. Run CacheSet.exe.
  2. Agree to the license terms.
  3. You can view the current cache size and adjust the minimum and maximum working set sizes, then click Apply.

Practical 6: Recovering and Inspecting Deleted Files

Aim: Using Autopsy to find, recover, and inspect deleted files from a forensic image.

  • Check for Deleted Files
  • Recover the Deleted Files
  • Analyze and Inspect the recovered files

Procedure:

In this practical, we will use Autopsy, a digital forensics platform, to analyze the forensic image created in Practical 1.

  1. Create a New Case: Open Autopsy and click on "New Case".
  2. Case Information: Give the case a name (e.g., "Deleted File Recovery") and choose a base directory to save the case files. Click Next.
  3. Optional Details: Fill in the case number and examiner details as needed. These can correspond to the details used when creating the image. Click Finish.
  4. Select Data Source: The "Add Data Source" wizard will open. Select "Disk Image or VM File" and click Next.
  5. Provide Path: Browse to and select the forensic image file (e.g., the `.dd` or `.E01` file from Practical 1). Click Next.
  6. Configure Ingest Modules: Leave the default ingest modules selected. These modules will automatically process the image to find deleted files, emails, web history, and more. Click Next.
  7. Start Analysis: Autopsy will show a summary. Click Finish to add the data source and begin the analysis. This may take some time depending on the size of the image.
  8. Check for Deleted Files: Once the analysis is complete, navigate the tree on the left side. Expand "Views" > "File Types" and click on "Deleted Files". The main window will now list all the deleted files Autopsy was able to identify.
  9. Recover/Extract Files: To recover a file, right-click on it in the list and select "Extract File(s)".
  10. Set Destination: Choose a directory on your system where you want to save the recovered files.
  11. Generate a Report: Go to the "Generate Report" button in the toolbar.
  12. Select Report Format: Choose the desired report format (e.g., Excel, HTML). Click Next.
  13. Select Results: Choose to include all results in the report and click Finish. Autopsy will generate a comprehensive report of its findings.
  14. View Report: Once finished, you can close the dialog and open the generated report from the directory where your case is saved.

Practical 7: Steganography Detection

Aim: Detect and extract hidden information or files within digital images using S-Tools.

  • Hide a secret file within a host image.
  • Detect and reveal the hidden content from the steganographic image.

Part A: Hiding Data

  1. Setup: Download and unzip the S-Tools utility. Run S-tools.exe.
  2. Prepare Files: Have a host image file (e.g., sample.bmp) and a secret file you want to hide (e.g., a text file) ready.
  3. Load Host Image: Drag the host image (sample.bmp) and drop it into the S-Tools window. An icon for the image will appear.
  4. Hide Secret File: Drag your secret file (e.g., the text document) and drop it directly onto the image icon *inside* the S-Tools window.
  5. Set Password: A dialog box will appear. Enter a password (e.g., "abc") and select an encryption algorithm (e.g., IDEA). Click OK.
  6. Save Steganographic Image: S-Tools will process the files. A new image icon will appear in the window, representing the image with the hidden data. Right-click this new icon, select "Save As," and save it with a new name (e.g., hidden_data.bmp). This new file will look identical to the original but contains your secret file.

Part B: Revealing Data

  1. Open Image: Open a new S-Tools window (or clear the existing one). Drag the steganographic image (hidden_data.bmp) into the window.
  2. Initiate Reveal: Right-click on the image icon inside S-Tools and select "Reveal".
  3. Enter Password: A dialog box will prompt you for the password and encryption algorithm. Enter the same credentials you used to hide the file (e.g., password "abc", algorithm IDEA) and click OK.
  4. View Revealed Archive: After processing, a "Revealed Archive" window will appear. This window lists the secret file(s) that were hidden inside the image.
  5. Save Secret File: Right-click on the file in the "Revealed Archive" list and select "Save As" to extract and save the hidden file to your computer. You can now open it to view the secret information.

Practical 8: Acquisition of Cell Phones and Mobile Devices

Aim: Perform a forensic analysis of a mobile device to retrieve call logs, text messages, and other data using MOBILedit Forensic.

  • Connect a mobile device to the forensic toolkit.
  • Perform a logical acquisition of the device data.
  • Analyze the retrieved data and generate a report.

Procedure:

  1. Setup: Download and install the MOBILedit Forensic toolkit.
  2. Start Connection: Launch the application. Click "Connect" to begin the acquisition process. Select "Phone" as the device type.
  3. Choose Connection Method: Select "Cable Connection" as the method for connecting to the phone.
  4. Enable USB Debugging: If the phone is not automatically detected, you must enable developer options on the device. Go to Settings > About Phone and tap the "Build Number" seven times. Then, go to the newly available Developer Options menu and enable USB Debugging.
  5. Connect and Scan: Connect the phone to the computer via USB cable. Allow the toolkit to scan for and identify the device. Once found, select it and click Next.
  6. Case Details: Fill in the case information (investigator name, case number, etc.) and browse for a directory to store the acquired data and logs.
  7. Select Acquisition Type: Choose the type of data you want to acquire. For a comprehensive analysis, select the option to acquire all data from the device.
  8. Begin Acquisition: Start the acquisition and wait for the process to complete.
  9. Analyze Data: Once the acquisition is finished, open the case file. The interface will allow you to browse the extracted data, such as:
    • Phonebook: Contacts stored on the device.
    • Call Logs: A record of all incoming, outgoing, and missed calls.
    • Messages: SMS and MMS messages.
    • Files: User files from internal storage, such as photos from the DCIM folder.
  10. Generate Report: Use the report generation feature to create a forensic report. Fill in the investigator details and select the desired output format, such as an Excel spreadsheet. A success message will appear once the report is created.

Practical 10: Web Browser Forensics

Aim: Analyze browser artifacts to reconstruct user activity using Browser History Examiner.

  • Analyze history, bookmarks, and download records.
  • Analyze cache and cookie data to reconstruct browsing history.
  • Extract and interpret timestamp data for user activity.

Procedure:

This practical uses Browser History Examiner. Note that this is a commercial tool with a free trial that limits the number of records you can view.

  1. Start the Application: Launch Browser History Examiner. If using the trial, click "Continue Trial" and "OK" on the subsequent prompts.
  2. Capture History: Navigate to the menu and select File > Capture History.
  3. Select Source: Choose to capture history from the local machine ("Capture from this PC") and click Next.
  4. Configure Capture: Select the web browser(s) you wish to analyze. Provide a destination directory where the extracted history files will be saved, then start the capture.
  5. Analyze Artifacts: Once the capture is complete, the tool's interface will populate with data. You can navigate through the different categories on the left pane to analyze various artifacts:
    • History: A chronological list of all visited URLs.
    • Bookmarks: Websites saved by the user.
    • Cache: View cached files, including images and full webpages, which can help reconstruct what the user saw.
    • Cookies: Inspect stored cookies for session data, user preferences, and tracking information.
    • Downloads: A record of all files downloaded by the browser.
    • Logins/Emails: May show email addresses and usernames used for website logins.
    • Searches: A specific list of search queries entered into search engines.
    • Website Visits: Provides a summary of user activity, often with timestamps indicating the last time a user was active online.
